# 电商购物平台开发流程文档

## 一、项目概述

本项目是一个基于 **Vue 3 + Vue Router + Pinia** 的单页面电商购物平台前端工程，采用 CDN 引入方式（无需构建工具），适合实验教学场景下的快速开发和演示。

### 技术栈
| 技术 | 版本 | 用途 |
|------|------|------|
| Vue 3 | 3.x (CDN) | 响应式 UI 框架，Composition API 支持 |
| Vue Router | 4.x (CDN) | 前端路由管理，Hash 模式 |
| Pinia | 2.x (CDN) | 全局状态管理（购物车） |
| localStorage | 浏览器原生 | 购物车数据持久化 |

### 阶段 2：配置 Pinia 购物车全局状态

#### 2.1 创建 Store
```javascript
const useCartStore = Pinia.defineStore('cart', { ... });
```

#### 2.2 State（状态）设计
| 字段 | 类型 | 说明 |
|------|------|------|
| `items` | Array | 购物车商品列表，从 localStorage 恢复 |
| `lastOrderAmount` | Number | 下单时的订单金额（清空购物车后仍可展示） |

每个 item 的结构：
```json
{
  "id": 1,
  "name": "无线蓝牙耳机",
  "price": 299,
  "image": "🎧",
  "color": "#2d3436",
  "quantity": 2
}
```

#### 2.3 Getters（计算属性）设计
| Getter | 说明 |
|--------|------|
| `totalCount` | 购物车商品总件数（含数量累加） |
| `totalPrice` | 购物车商品总价 |
| `isEmpty` | 购物车是否为空 |

#### 2.4 Actions（操作）设计
| Action | 说明 |
|--------|------|
| `_persist()` | 私有方法：将购物车数据写入 localStorage |
| `addToCart(product, quantity)` | 添加商品到购物车，若已存在则累加数量 |
| `removeFromCart(productId)` | 从购物车移除指定商品 |
| `updateQuantity(productId, quantity)` | 更新商品数量（≤0 时自动移除） |
| `increaseQty(productId)` | 数量 +1 |
| `decreaseQty(productId)` | 数量 -1（减至 0 自动移除） |
| `checkout()` | 下单结算：保存订单金额，清空购物车 |
| `clearCart()` | 清空购物车 |

#### 2.5 数据持久化
- 每次操作后调用 `_persist()` 将 `items` 写入 `localStorage`
- 初始化时从 `localStorage` 恢复数据：`JSON.parse(localStorage.getItem('cart_items') || '[]')`

### 阶段 4：设计路由与页面跳转

#### 4.1 路由表设计
| 路径 | 组件 | 说明 |
|------|------|------|
| `/` | HomePage | 首页（Hero + 热销商品） |
| `/products` | ProductsPage | 全部商品列表 |
| `/product/:id` | ProductDetail | 商品详情（动态路由） |
| `/cart` | CartPage | 购物车 |
| `/order-success` | OrderSuccessPage | 下单成功页 |
| `/:pathMatch(.*)*` | NotFoundPage | 404 页面（全局通配） |

路由模式：`createWebHashHistory()`（Hash 模式，兼容静态部署）。

#### 4.2 导航栏路由链接
使用 `<router-link>` 组件实现首页、全部商品、购物车的导航切换，购物车图标旁显示商品数量徽章。

### 阶段 5：实现页面组件

#### 5.1 首页（HomePage）
- **Hero 区域**：活动 banner + 抢购按钮 → 跳转商品列表
- **服务特性**：极速配送、正品保障、专属客服三栏展示
- **热销爆款**：展示前 4 件商品（复用商品卡片组件模式）
- 每件商品支持「查看详情」和「加入购物车」操作

#### 5.2 全部商品页（ProductsPage）
- 展示全部 8 件商品的网格布局
- 使用 `auto-fill + minmax(200px, 1fr)` 实现自适应列数
- 悬停效果：卡片上浮 + 阴影加深

#### 5.3 商品详情页（ProductDetail）
- 通过 `$route.params.id` 获取商品 ID
- 展示商品大图、名称、描述、价格
- 「加入购物车」按钮 + 添加成功反馈（文字 + 按钮禁用 2s）
- 商品不存在时的提示页

#### 5.4 购物车页（CartPage）
- **空态**：显示空购物车提示 + 引导去购物
- **列表态**：每行展示商品图片、名称、单价、数量调节器（±）、小计、删除按钮
- **底栏汇总**：总件数 + 合计金额
- **操作按钮**：🗑️ 清空购物车（带确认弹窗）、💳 立即结算

#### 5.5 下单成功页（OrderSuccessPage）⭐
- 动画勾号（CSS `@keyframes popIn`）
- 展示订单编号（时间戳生成）、实付金额
- **10 秒倒计时自动跳转首页**
- 手动按钮：「返回首页」「继续购物」

#### 5.6 404 页面（NotFoundPage）⭐
- 大号 404 状态码
- 友好提示文案
- **10 秒倒计时自动返回首页**
- 「返回首页」手动按钮

### 阶段 6：实现 Toast 消息通知系统

#### 6.1 事件总线模式
```javascript
const toastBus = {
  listeners: [],
  emit(msg) { this.listeners.forEach(fn => fn(msg)); },
  on(fn) { this.listeners.push(fn); }
};
```

#### 6.2 Toast 显示
- 固定在页面右上角（`position: fixed`）
- 消息从右侧滑入（CSS `@keyframes slideIn`）
- 自动消失（2.5s 后移除）
- 支持多条消息堆叠

## 三、关键技术要点

### 3.1 路由跳转方式

| 场景 | 实现方式 | 说明 |
|------|----------|------|
| 导航栏切换 | `<router-link to="/cart">` | 声明式导航 |
| 编程式跳转 | `this.$router.push('/order-success')` | 下单后跳转成功页 |
| 自动跳转 | `setInterval + $router.push('/')` | 404/成功页倒计时跳转 |
| 404 兜底 | `/:pathMatch(.*)*` | 匹配所有未定义路由 |
| Hash 模式 | `createWebHashHistory()` | URL 以 `#` 分隔 |

### 3.2 购物车核心逻辑

```
添加商品流程:
  用户点击「加入购物车」→ addToCart(product)
    ├─ 商品已在购物车? → quantity += 1 → 提示「已更新数量」
    └─ 商品不在购物车? → push 新 item → 提示「已加入购物车」
    → _persist() 写入 localStorage
    → Toast 通知用户

删除商品流程:
  用户点击 ✕ → removeFromCart(id)
    → 查找并 splice 移除 → _persist() → Toast 通知

下单结算流程:
  用户点击「立即结算」→ checkout()
    → lastOrderAmount = totalPrice → items = [] → _persist()
    → $router.push('/order-success')
    → 成功页展示订单金额 + 倒计时返回

清空购物车:
  用户点击「清空购物车」→ confirm 确认 → clearCart()
    → items = [] → _persist() → Toast 通知
```

### 3.3 数据持久化策略
- **存储介质**：`localStorage`
- **存储 Key**：`cart_items`
- **存储格式**：JSON 字符串
- **时机**：每次增/删/改/清空后立即写入
- **恢复**：应用初始化时从 localStorage 读取

## 六、总结

本项目完整实现了一个电商购物平台的前端核心功能，涵盖：

- ✅ **路由导航**：Vue Router 管理 6 个页面路由，含动态路由和 404 通配
- ✅ **404 页面**：10 秒倒计时自动跳转首页，带手动返回按钮
- ✅ **下单成功页**：订单信息展示 + 10 秒倒计时自动跳转
- ✅ **Pinia 状态管理**：购物车全局状态，State/Getters/Actions 完整定义
- ✅ **购物车 CRUD**：添加（含重复累加）、删除、数量调节、总价计算
- ✅ **清空购物车**：确认弹窗 + 一键清空
- ✅ **下单结算**：保存订单金额 → 清空购物车 → 跳转成功页
- ✅ **Toast 通知**：事件总线驱动的消息提示系统
- ✅ **数据持久化**：localStorage 自动读写，刷新不丢失
- ✅ **响应式布局**：CSS Grid/Flexbox + 媒体查询，适配移动端
